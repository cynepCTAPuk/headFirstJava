Протокол HTTP

Построение URL для произвольного сетевого ресурса

          
  • scheme — протокол передачи данных (http, https, ftp etc.)
  • host — наименование сервера в интернете (можно указать явно ip 192.168.0.1, localhost)
  • port — порт, на котором расположен данный сетевой ресурс, для http по дефолту равен 80
  • path — путь к сетевому ресурсу внутри сервера
  • parameters — параметры, передаваемые вместе с запросом
  • type request — в браузере не указывается, всегда подразумевается GET

Структура протокола HTTP. Запрос

Запрос представляет собой обычный текст, состоящий из следующих строк:

строка http запроса формат
стартовая строка метод URI HTTP/версия
заголовки название заголовка: значение
пустая строка отделяет заголовки от тела запроса
тело запроса  зависит от значения заголовка Content-Type

Пример типичного запроса с заголовком и телом:

POST /api/client-token HTTP/1.1
Host: www.test.edu
Content-Type: application/json
cache-control: no-cache

{
 "seed": 298122,
 "mode": "y"
}

Структура протокола HTTP. Ответ

Ответ представляет собой обычный текст, состоящий из следующих строк:

часть http ответа формат
стартовая строка HTTP/версия статус описание
заголовки название заголовка: значение
пустая строка отделяет заголовки от тела ответа
тело ответа  зависит от значения заголовка Content-Type

Пример типичного ответа с заголовком и телом:

HTTP/1.1 200 ok
Content-Type: application/json;charset=UTF-8
Date: Thu, 21 Nov 2019 08:35:08 GMT

{
    "token": "eyJ2ZXIiOjEsInR5cCI6IkpXVCIsInNidCI6",
    "type": "Bearer",
    "expiresIn": 51400
}
