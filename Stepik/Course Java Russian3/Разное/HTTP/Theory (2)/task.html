REST

REST API — это один из самых популярных протоколов передачи данных, реализованный на базе протокола HTTP.

Для описания сетевого ресурса, как правило, используется формат json (реже xml).

{
  "id": 10000011,
  "first_name": "Иван",
  "last_name": "Иванов",
  "phone": "123123",
  "address": {
    "street": "ул. ленина, д. 4",
    "city": "москва"
  }
}

Рассмотрим типичные rest-операции.

Получение списка всех сотрудников

запрос:

GET http://www.test.edu/employee/

ответ:

HTTP/1.1 200 ok
Content-Type: application/json

[
  {
    "id": 10000011,
    "first_name": "Иван",
    "last_name": "Иванов",
    "phone": "123123",
    "address": {
      "street": "ул. ленина, д. 4",
      "city": "москва",
      "zip": "612613"
    }
  },
  {
    "id": 10000012,
    "first_name": "Петр",
    "last_name": "Петров",
    "phone": "123123",
    "address": {
      "street": "ул. ленина, д. 4",
      "city": "москва",
      "zip": "612613"
    }
  }
]

Получение одного сотрудника

запрос:

GET http://www.test.edu/employee/10000011

ответ:

HTTP/1.1 200 ok
Content-Type: application/json

{
  "id": 10000011,
  "first_name": "Иван",
  "last_name": "Иванов",
  "phone": "123123",
  "address": {
    "street": "ул. ленина, д. 4",
    "city": "москва"
  }
}

Создание нового сотрудника

запрос:

POST http://www.test.edu/employee/10000017
Content-Type: application/json

{
  "id": 10000017,
  "first_name": "Маргарита",
  "last_name": "Иванова",
  "phone": "123777",
  "address": {
    "street": "ул. ленина, д. 11",
    "city": "москва"
  }
}

ответ:

HTTP/1.1 201 created

Удаление сотрудника

запрос:

DELETE http://www.test.edu/employee/10000017

ответ:

HTTP/1.1 200 ok
