Старое API vs новое API

Старое API:

  • java.util.Date – по сути long, где записано время в миллисекундах, ноль соответствует 1 января 1970 года
  • java.util.Calendar – универсальный рембо класс, который включает в себя экземпляр Date, а также информацию о временных зонах

Date, как и Calendar, имеет ужасное название, не отражающее суть его поведения. Оба этих класса не иммутабельны.

В настоящее время очень много старого кода написано с применением java.util.Date.

При работе со временем, если явно не указывать временную зону, она берется по умолчанию. Например, для жителей Москвы разрабатывали приложение в Москве, разработчики тестировали систему в Москве, запустили конечный продукт в облаке в США – разница будет 12 часов. 

Новое API:

  • java.time.Clock – часы, могут быть не идущими, опаздывающими, идущими с различной степенью точности
  • java.time.ZoneId – обозначает временную зону
  • java.time.Instant – аналог java.util.Date, ничего не знает про временную зону, toString() вернет время в формате ISO-8601
  • java.time.LocalTime – это кортеж (час, минуты, секунды, наносекунды)
  • java.time.LocalDate – это кортеж (год, месяц, день месяца)
  • java.time.LocalDateTime – оба кортежа вместе

Для новых классов практически нет ограничения по времени:

LocalDateTime.MIN.toInstant(ZoneOffset.UTC); // -999999999-01-01T00:00:00Z
LocalDateTime.MAX.toInstant(ZoneOffset.UTC); // +999999999-12-31T23:59:59.999999999Z

Полезные enum и классы:

  • java.time.DayOfWeek – перечисление с днями недели, можно получить названия дней недели на русском
  • java.time.Month – перечисление с месяцами года, можно получить названия месяцев на русском
  • java.time.format.DateTimeFormatter – служит для преобразования даты в текст и обратно
  • java.time.format.ZoneName – не публичный класс, в нем, например, можно посмотреть, как правильно пишется название временной зоны для Москвы или для Минска 

В DateTimeFormatter уже есть много предопределенных форматеров, например:

Форматер Пример
BASIC_ISO_DATE 20191001
ISO_DATE_TIME 2019-10-01T18:04:15.5671387
ISO_DATE 2019-10-01
ISO_TIME 18:04:15.5671387
ISO_WEEK_DATE 2019-W40-2 // 2019 год, 40-я неделя, вторник
ISO_ZONED_DATE_TIME 2011-12-03T10:15:30+01:00[Europe/Paris]
ofPattern("yyyy-MM-dd'T'HH:mm:ss") 2019-10-02T11:03:00
