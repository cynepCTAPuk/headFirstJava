Домашний проект — Гипотетическая децентрализованная система

Как Вы понимаете, многопоточность — гораздо более обширная тема. Для более детального погружения в тематику предлагаю выполнить домашний проект. Проверятся на степике проект не будет.

Описание задания:

Вам дана гипотетическая децентрализованная система, каждый компонент которой является микросервисом, а общение между компонентами инфраструктуры происходит по API.

Ваша задача: создать прототип микросервиса, обеспечивающий доставку сообщений микросервисам, входящим в состав системы. Микросервис является брокером. Его задача — получать сообщения от внешних источников и доставлять их своим подписчикам.

Приложение должно обладать следующими возможностями:

  • подписка/отписка на topic (если такого топика еще нет, то он создается),
  • доставка полученного сообщения всем подписчикам topic-ов, указанных в сообщении.

Сообщение приходит от абстрактного внешнего источника. Фактическая реализация приема сообщений откуда-то извне не требуется, сообщения можно генерировать случайным образом. Подписчиков может быть много (тысячи), подписываются и отписываются они постоянно. Сообщений тоже много (сотни в единицу времени). Реализовывать полноценную работу с сетью не требуется, достаточно простого интерфейса для взаимодействия подписчиков с разрабатываемым микросервисом.

Структура и формат сообщений, а также потребность в дополнительных параметрах остаётся на Ваше усмотрение.

Каждый микросервис должен "жить" в отдельном потоке. Брокер должен "жить" в отдельном потоке. Подписка/отписка микросервиса на/от брокера должна осуществляться через метод брокера.

Вам нужно придумать, как протестировать работоспособность Вашего брокера в условиях многопоточности. Если для этого Вы будете использовать вывод в консоль, убедитесь, что отправка логов в консоль не будет тормозить обмен сообщениями между микросервисами.

Для реализации проекта разрешается использовать только те средства, которые есть в java core.

Критерии успеха:

  • каждую секунду общее количество сообщений, поступающих в брокера, измеряется тысячами
  • все подписанные на topic-и микросервисы получают сообщения без потерь
  • надежность отправки/получения сообщений в системе подтверждается программным путём
