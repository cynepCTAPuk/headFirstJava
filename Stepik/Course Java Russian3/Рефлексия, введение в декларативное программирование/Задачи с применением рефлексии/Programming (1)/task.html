Контейнер операций

Необходимо реализовать функцию:

  • на вход функция получает список объектов (пул объектов);

  • в некоторых объектах будут встречаться методы с аннотацией @BotRequestMapping;

  • в аннотации @BotRequestMapping представлен url некоторой операции;

  • необходимо сформировать HashMap с операциями, где ключом будет url операции, а значением — сама операция;

  • в дальнейшем вызов операций будет производиться произвольно, без Вашего участия;

  • при вызове операции по url должен вызываться метод с таким же url;

  • гарантируется, что все методы будут без аргументов;

  • гарантируется, что каждый класс представлен единственным объектом.

Пример тестового пула объектов (case 1):

class A {
        @BotRequestMapping("hello")
        void methodA1() {
            System.out.println("hello world!!!");
        }

        @BotRequestMapping("goodbye")
        void methodA2() {
            System.out.println("goodbye my friend!!!");
        }

        void methodA3() {
            System.out.println("bad call!!");
        }
    }

Sample Input:
case 1
Sample Output:
hello world!!!
hello world!!!
goodbye my friend!!!

Sample Input:
case 2
Sample Output:
hello world!!!
madam I'm adam
A cat may look at a king
A barking dog never bites
goodbye my friend!!!
