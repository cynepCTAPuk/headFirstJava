Dependency Injection (DI)

Необходимо реализовать функцию:

  • на вход функция получает список объектов (пул объектов);

  • гарантируется, что каждый класс представлен единственным объектом;

  • в объектах могут встречаться поля, помеченные аннотацией @Autowired;

  • для полей со стереотипом @Autowired Вам нужно выбрать из пула объектов подходящий объект по типу и засетить его;

  • если подходящий объект по типу не найден, то нужно бросить исключение CandidateNotFindException.

Пример тестового пула объектов (case 1):

class A {
        @Autowired
        B b;

        C c;

        public B getB() {
            return b;
        }

        public void setB(B b) {
            this.b = b;
        }

        public C getC() {
            return c;
        }

        public void setC(C c) {
            this.c = c;
        }
    }

    class B {
        @Autowired
        C c;

        public C getC() {
            return c;
        }

        public void setC(C c) {
            this.c = c;
        }
    }

    class C {

    }
