Аннотации

Аннотации — это некоторая дополнительная информация о классе, методе, поле и т.д.

В некоторых источниках аннотации называют стереотипами.

Для создания нового стереотипа, как правило, применяют стандартные аннотации:

  • @Target
  • @Retention
  • @Documented
  • @Inherited

Пример создания новой аннотации:

@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
public @interface MyFirstAnnotation {

}

Пример использования аннотации:

@MyFirstAnnotation 
public class MyClass {

}

Жизненный цикл аннотаций:

  • доступны только в исходниках (доступно для IDE, инструментов кодогенерации типа Lombok);
  • доступны в исходниках и в байт-коде (доступно для внешних инструментов работы с байт-кодом);
  • доступны в исходниках, в байт-коде и во время исполнения (может использоваться в рефлексии).
